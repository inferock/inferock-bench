# covrun Phase 0 - Code-derived Measure Preconditions

This is a package-local sanitized fixture used to derive baseline placeholders in tests. It documents code-derived measure preconditions from a historic covrun; it is not a public incidence estimate and not live receipt evidence.

| What to use it for | Boundary |
| --- | --- |
| Understanding which code paths open coverage surfaces | OK as a fixture-backed method reference. |
| Deriving bootstrap-required baseline placeholders in tests | OK; the fixture lacks per-task token samples. |
| Public proof of current provider behavior | Not allowed; use real measured runs and receipts. |

Read first:
- Public coverage methodology, especially the normal-usage rule: tests carry ordinary preconditions without manufacturing failures.
- Public local key-handling docs, especially provider-key and local `ibl_` boundaries.

## Shipped bench `measures` rows

The installed bench summary ships 13 rows from `measureRows()`.

| rowKey | measure label | Code path | Exact code precondition for non-vacuous/watchable evidence | What moves bench row out of `not_exercised` | Normal config/traffic that can carry it | Code-gated or missing surface |
|---|---|---|---|---|---|---|
| `broken_output` | Broken output: bad JSON/schema | `detectBenchJsonModeBrokenOutput()` plus `detectBrokenOutput()` | Bench JSON mode gate: `event.request.generation.responseFormat` or `response_format` is a string containing `json`, or object `{type:"json_object"}` / `{type:"json_schema"}`. The bench-specific JSON-mode detector only runs when `event.meta.outputSchemaVersion` is absent, provider refusal/content-filter is absent, content is non-empty, and the response is invalid JSON. The library schema detector needs `event.meta.outputSchemaVersion`, a registered output schema for `(tenantId, schemaVersion)`, no provider refusal/content-filter, then JSON parse and AJV validation. | `measureRow(... exercised: events.some(jsonModeRequested))`, so one JSON-mode request is enough for a clean watched row; a broken JSON response emits `BROKEN_OUTPUT`. | OpenAI Chat/Responses request with `response_format: {type:"json_schema", ...}` or `{type:"json_object"}` and ordinary extraction prompts. Anthropic native structured output is not exposed as a schema contract here; Anthropic tool-schema extraction exercises the separate tool detector, not this row. | Real schema registry path is not exposed by `inferock-bench` config/proxy. `outputSchemaVersion` exists on adapter input but the bench proxy never sets it and no bench config registers schemas. JSON mode remains openable for OpenAI via request body capture. |
| `anthropic_token_crosscheck` | Anthropic output-token recount | `detectBenchAnthropicTokenCrosscheck()` -> `crossCheckAnthropicOutputTokens()` | `event.request.provider === "anthropic"`. Cross-check uses verified per-model count_tokens calibration when available, otherwise bench fallback bound `chars*6 + 16`. | `measureRow(... exercised: events.some(provider==="anthropic"))`; any Anthropic call is watched clean unless a verified recount delta or fallback gross-bound violation emits `ANTHROPIC_TOKEN_CROSSCHECK`. | Any normal Anthropic Messages call, including Claude Code traffic through `ANTHROPIC_BASE_URL`. | Bench summary has no live count_tokens client; reconciliation can add verified count_tokens samples. Fallback mode uses `bench_default_count_tokens_unavailable`. |
| `duplicate_request_id` | Duplicate request-ID events | `duplicateRequestIdSignals()` -> `buildDuplicateRequestIdSignal()` plus `duplicateRequestIdOpenability()` | Events are grouped by `event.request.tenantId`, `event.request.provider`, `event.request.requestId`. Clean openability requires captured operation/idempotency evidence: `operationId`, `bodyHash`, or `retryCorrelationId`. | A duplicate group moves the row to `signal`; otherwise operation/body-hash/retry-correlation evidence marks `watched_clean`. | Normal SDK or application traffic that carries operation, body-hash, or retry-correlation evidence. Let SDK-native retry/concurrency produce any duplicate naturally; do not hand-send a duplicate. | Plain unique request IDs alone do not open clean duplicate-ID coverage; the precondition is logical operation/idempotency evidence. |
| `provider_latency_slo` | Provider latency (Inferock default SLA) | `runStatelessDetectors()` -> `detectLatencyBilled()` with `defaultLatencySloPolicyForEvent()` | Any normalized event with finite `event.timing.latencyMs > 0` and provider-billed tokens opens the default latency evaluation. Provider-recognized/disclosed-SLO latency additionally requires provider-only timing evidence: `providerRequestStartedAt`, `providerResponseEndedAt`, and finite `providerElapsedMs`. Latency loss requires `evaluateDefaultLatency(event).exercised`, `excessMs > 0`, billed tokens, and not provider downtime. | Any measured call moves row to `exercised`; slow default-standard calls emit `LATENCY_BILLED`. Bench watched evidence reports total-latency openability and provider-elapsed timing availability separately. | Automatic on all normal calls. Streaming calls add first-event/first-token/chunk timing and select interactive latency segments. Proxy fetch-boundary timing can report provider elapsed separately from gateway overhead when the response body or stream reaches a terminal boundary. | `LATENCY_DETECTOR_ENABLED=false` disables the detector in `stateless.ts`; default is enabled. Missing provider-elapsed timing does not block the Inferock-standard default row, but it keeps provider-recognized/disclosed-SLO attribution unavailable. |
| `tool_call_validity` | Tool-call validity | `runStatelessDetectors()` -> `detectToolCallValidity()` plus `toolCallValidityOpenability()` | Request `tools` are captured as `request.toolDeclarations` for OpenAI Chat, OpenAI Responses, Anthropic Messages, Gemini, and OpenAI-compatible chat where the adapter supports tool schemas. Response tool calls are captured from `response.rawToolCalls` / `response.toolCalls`; Responses requires top-level `type:"function_call"` items with `name` and JSON-string `arguments`, not Chat `function.arguments`. Signals require malformed tool calls, schema-invalid arguments, undeclared tool use, tool-choice violation, or tool-stop reason without tool block. | Invalid tool use moves the row to `signal`; otherwise same-surface tool declarations plus response tool evidence mark that provider surface watched-clean. Bench details identify Chat, Anthropic, Gemini, OpenAI-compatible chat, and Responses separately so Chat/Anthropic traffic cannot silently open the Responses sub-surface. | Normal tool/function-call traffic from the `tool_schema_plan` task or equivalent application flows. Responses row-12 coverage needs ordinary `/v1/responses` traffic that declares function tools and returns top-level `function_call` output. | The clean state requires both request declarations and response tool evidence on the same supported surface. Declarations without a provider tool-use response are not enough. Responses structured-output traffic without tools/function_call remains not-openable for row 12, not watched-clean. |
| `security_governance` | Security/governance overlay | `runSecurityDetectors()` plus `securityOpenability()` in `summary.ts` | Detector runs over output text/tool-call string leaves and provider safety entries. Signals require either an exact secret-like match in provider output or any `response.providerSafety` entry. Clean passive coverage opens when the normal `organic_safety_overlays` suite task ran, request security capture is marked complete, or provider safety evidence is captured. Request secret attribution needs optional `REQUEST_SECRET_DIGEST_KEY_ID` + `REQUEST_SECRET_DIGEST_KEY`, but passive openability does not. | Signals mark `signal`; otherwise passive precondition evidence marks `watched_clean`. Missing passive evidence marks `not_openable`. | Organic only. Ordinary tasks should not include or solicit secrets or safety events; provider safety entries, if any, are captured passively. The `organic_safety_overlays` task is neutral normal traffic. | No manufactured secrets, leaks, adversarial prompts, or policy-triggering content are allowed to open this surface. |
| `openai_content_filter` | Content-filter overlay | `runContentFilterOmittedOutputDetectors()` plus `contentFilterOpenability()` | Signals require OpenAI provider content-filter evidence: `response.providerSafety` entry `{source:"provider", kind:"content_filter"}` or `response.finishReason === "content_filter"`. Clean openability requires inspected OpenAI completion traffic with `expectCompletion !== false` and status `<500`. | Content-filter evidence marks `signal`; otherwise inspected OpenAI completion traffic marks `watched_clean`. | Organic only. Normal OpenAI calls pass through detector; no provocation should be used. | No adversarial or policy-triggering prompts are allowed to open this surface. |
| `stream_termination_evidence` | Stream-termination overlay | `runStreamTerminationDetectors()` plus `streamTerminationOpenability()` | Requires schema v2 and stream timing evidence. Clean openability requires terminal evidence such as `chunkCount > 0`, `firstEventAt`, `firstContentDeltaAt`, or `lastChunkAt` plus complete terminal status. Signals are emitted only for terminal anomalies: Anthropic post-200 stream error, missing/unknown terminal status, or aborted stream with attribution. | Terminal anomalies mark `signal`; otherwise terminal stream evidence marks `watched_clean`. | Normal streaming requests (`stream: true`) through OpenAI/Anthropic. | A stream request without terminal evidence remains not-openable for this surface. |
| `retry_amplification` | Retry amplification | `runRetryAmplificationDetectors()` plus `retryAmplificationOpenability()` | Requires schema v2 and retry observation. In-call gate is `x-stainless-retry-count` request header parsed as integer > 0 or captured retry attempts (`attempt.status === "retry"`, `retryReason`, or `finalSelected === false`). Provider-directed attribution can use `retry-after`, `retry-after-ms`, truthy `x-should-retry`, or 5xx status on response/attempt headers. | Retry amplification evidence marks `signal`; otherwise SDK/native retry-observation evidence marks `watched_clean`. | Use normal provider SDKs with default retry behavior and concurrency; bench captures `x-stainless-retry-count` from inbound request headers. Do not manufacture retries. | No hand retries. The surface opens only when SDK/native retry observation is present. |
| `served_model_mismatch` | Served model mismatch | `runStatelessDetectors()` -> `detectServedModelMismatch()` plus `servedModelOpenability()` | Requires requested model and provider-response served-model identity. A mismatch signal needs served model evidence that differs from the requested model outside documented alias handling. | Mismatch marks `signal`; otherwise provider-response served-model identity marks `watched_clean`. | Normal provider responses carry served model identity for supported routes. | Missing provider-response model identity is not-openable. |
| `cache_integrity` | Cache integrity | `cacheIntegrityOpenability()` in `summary.ts`; library helpers `buildCacheRateAnomalySignal()` and `buildCacheDiscountAtRiskSignal()` define the charge anomaly and usage/pricing at-risk semantics. | Coverage requires at least two repeated shared-prefix calls and cache-token evidence. `CACHE_DISCOUNT_AT_RISK` needs usage and pricing only; `CACHE_RATE_ANOMALY` charge reconciliation additionally requires matching provider charge observation. Cache billed usage means `event.usage.cache.read > 0`, `usage.cache.creation > 0`, or usage categories with positive `cache_creation`, `cached`, or `cache_read`. | Any cache at-risk or anomaly signal marks `signal`. Otherwise repeated shared-prefix precondition + cache tokens + hosted/imported or configured charge observation marks `watched_clean` when no anomaly is emitted. Missing charge data is `not_openable` only when no cache at-risk signal fired and charge reconciliation cannot run. | Normal repeated long-prefix calls can produce provider cache usage fields, especially OpenAI cached tokens or Anthropic cache read/creation tokens. Hosted/imported provider billing data or a configured charge-observation file is required only for charge reconciliation. | Local BYOK traffic without provider charge observation cannot honestly open the charge-reconciliation surface unless the usage+pricing cache discount-at-risk signal fires. |
| `drift_regression` | Drift / regression | `driftRegressionOpenability()` in `summary.ts`, plus coverage runner per-model canary overlay | Opens on a completed per-model canary against an established baseline segment, or on a configured drift replay contract with 3-5 completed same-window repeats and matcher evidence within threshold. | A flagged canary emits drift loss rows through the coverage runner; otherwise an established clean canary or replay contract marks `watched_clean`. Baseline collection remains `not_openable`. | Normal known-answer canary traffic or normal repeated prompts carrying an explicit drift replay contract. | Repeated prompts without a contract or canary baseline are not enough. Protocol changes start a new canary baseline segment. |
| `factuality` | Factuality overlay | `benchFactualitySignals()` -> `runFactualityDetectors()` plus `factualityOpenability()` | Requires captured `request.factualityContract` with known-answer mode and response content. Signals require a known-answer contradiction. | A contradiction marks `signal`; otherwise a captured factuality contract marks `watched_clean`. | Normal known-answer tasks that legitimately carry an expected-answer contract. | Known-answer prompts without a captured contract remain not-openable. |

## Config gates found

| Gate | Default | Opens/closes |
|---|---:|---|
| `DOWNTIME_DETECTOR_ENABLED` | enabled unless exactly `false` | Provider downtime stateless detector. |
| `LATENCY_DETECTOR_ENABLED` | enabled unless exactly `false` | Latency detector and loss rows; bench latency row still computes details from events. |
| `OPENAI_TOKEN_RECOUNT_ENABLED` | enabled unless exactly `false` | OpenAI token recount detector, but no bench measure row. |
| `INFEROCK_BENCH_OPENAI_API_KEY` / `OPENAI_API_KEY` or persisted config | off unless set | OpenAI traffic through bench. |
| `INFEROCK_BENCH_ANTHROPIC_API_KEY` / `ANTHROPIC_API_KEY` or persisted config | off unless set | Anthropic traffic through bench. |
| `REQUEST_SECRET_DIGEST_KEY_ID` + `REQUEST_SECRET_DIGEST_KEY` | off unless set | Security output secret attribution to request context; not needed for passive provider-safety overlay. |
| Known-answer/factuality contract config | present in coverage-suite annotations | Opens factuality only when the captured request carries the known-answer contract. |
| Cache charge observation config | absent by default | Opens cache charge reconciliation only when hosted/imported provider billing data or a charge-observation file is available. Local BYOK runs without charge data can still emit `CACHE_DISCOUNT_AT_RISK` from usage and pricing alone. |
| Drift baseline/replay config | absent by default | Opens drift/regression only through a completed canary baseline segment or a configured replay contract. |

## Traffic implications for this covrun

- Open with ordinary traffic: JSON mode, Anthropic token cross-check, latency, tool-call validity, stream timing evidence, operation/idempotency evidence, retry observation, served-model identity, passive security via `organic_safety_overlays`, and provider safety/content-filter overlays by detector presence.
- Send but report not-openable honestly when preconditions remain missing: cache traffic with no at-risk signal and no hosted/imported provider charge observation, drift traffic without a completed canary baseline or replay contract, and known-answer traffic without a captured factuality contract.
- Do not manufacture signals: no max-token truncation, no adversarial/malformed prompts, no hand-crafted duplicate request IDs, no content-filter provocation.
